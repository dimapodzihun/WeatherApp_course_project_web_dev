# 🌦 Курсовий проєкт 2 курс, 2 семестр
## 📚 Дисципліна: Технології веб-розробки

## 📌 Тема
Розробка вебдодатку для аналізу та прогнозування погодних умов  
на основі даних з супутників та метеорологічних станцій.

---

## 🧩 Постановка задачі
Розробити вебсистему для:
- збору метеорологічних даних
- аналізу погодних умов
- прогнозування змін погоди

---

## 🚀 Можливості системи
- 🌍 Отримання погодних даних у реальному часі
- 📈 Короткострокове та довгострокове прогнозування
- 📊 Візуалізація кліматичних змін
- 🕓 Аналіз історичних погодних трендів
- ⚡ Інтерактивні графіки

---

## 🛠 Технології
- 🐍 Python (Flask)
- 🐘 PostgreSQL
- 🌐 HTML
- 🎨 Tailwind CSS
- ☁️ OpenWeather API
- 📊 Pandas
- ⚡ HTMX
- 📉 Python (Plotly)

  ---

  ## ✅ Повний перелік функціональних можливостей проєкту

1. Реєстрація нового користувача з перевіркою коректності email.
2. Авторизація користувача в системі.
3. Вихід із облікового запису.
4. Видалення облікового запису користувача.
5. Пошук міста за назвою.
6. Отримання підказок під час введення назви міста.
7. Отримання поточної погоди для вибраного міста.
8. Відображення температури, вологості, тиску, швидкості вітру та опису погоди.
9. Автоматичне завантаження головного погодного блоку для збереженого міста.
10. Збереження міст у персональний список користувача.
11. Перегляд усіх збережених міст.
12. Оновлення погодних даних для окремого збереженого міста.
13. Видалення міста зі списку збережених.
14. Автоматичне збереження історії погодних показників для міст.
15. Формування погодної картки міста на основі історичних даних.
16. Побудова короткострокового прогнозу погоди на 5 днів.
17. Перегляд погодинного прогнозу на конкретну дату.
18. Відображення денного та погодинного прогнозу для вибраного міста.
19. Генерація довгострокового прогнозу на 14 днів.
20. Генерація оцінювального прогнозу на 30 днів.
21. Аналіз зміни температурного тренду в довгостроковому прогнозі.
22. Персоналізовані рекомендації відповідно до поточної погоди.
23. Формування погодних попереджень про сильний вітер, опади, зниження температури та інші ризики.
24. Оцінка комфортності погодних умов.
25. Використання rule-based логіки для визначення рівня комфорту.
26. Використання ML-моделі для прогнозування комфортності погоди за наявності достатньої історії.
27. Порівняння поточної погоди у двох містах.
28. Порівняння прогнозу погоди для двох міст.
29. Генерація текстового підсумку порівняння міст.
30. Побудова графіка порівняння температур між містами.
31. Побудова графіка порівняння вологості між містами.
32. Побудова графіка порівняння поточних погодних показників.
33. Відображення аналітики за історичними даними міста.
34. Формування зведення про зміни температури, вологості та тиску.
35. Визначення кліматичних відхилень на основі накопиченої історії.
36. Побудова графіка історії температури.
37. Побудова графіка зміни вологості й тиску.
38. Побудова графіка кліматичних змін і відхилень.
